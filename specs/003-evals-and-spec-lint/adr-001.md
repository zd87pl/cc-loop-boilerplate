# ADR-001: Deterministic-only evals; lint before the model

| Field    | Value             |
| -------- | ----------------- |
| Status   | accepted          |
| Date     | 2026-06-24        |
| Deciders | platform-dx       |
| Related  | SPEC-003, PRD-003 |

## Context

We want to assert the loop's guarantees in CI without paying model cost or
introducing flakiness, and we want cheap mechanical spec checks separated from
expensive model judgment.

## Decision

We will:

1. Keep the **CI eval suite deterministic** — no model calls. It asserts the
   guardrails that are deterministic (halt-on-ambiguity, spec-lint, secret /
   destructive veto, protected-branch refusal, a real stack gate). Model-
   dependent guardrails (ADR-conflict, drift) are scaffolded as **live-only**
   fixtures, run with `make loop`, never in CI.
2. Run a **deterministic `spec-lint` gate before `/spec-review`** — mechanics
   (structure, ids, coverage, placeholders) are caught for free; the model is
   reserved for judgment. Both the lint gate and the clarification gate halt with
   `needs_clarification`.
3. Detect placeholder/clarification markers as **line-leading markers**, not
   inline mentions, so a spec *about* TODOs or clarifications is not falsely
   flagged (the same self-reference care as the secret guard).
4. Ship a **real reference implementation** (Python, stdlib-only) so the gates
   execute and the verifier has concrete code to trace.

## Consequences

**Positive**

- Loop changes are caught by assertions; specs get instant cheap feedback;
  the example proves end-to-end behavior.

**Negative / trade-offs**

- The deterministic suite can't cover model-judgment guardrails (hence live
  fixtures).
- `spec-lint` is heuristic on markdown tables; unusual spec formats may need
  tuning.

**Neutral**

- The reference example is Python; other languages can be added the same way.

## Alternatives considered

- **Run the full loop live in CI** — rejected: token cost + flakiness on every PR.
- **Only model-based spec review (no lint)** — rejected: pays a model to catch a
  missing table cell; determinism is cheaper and more reliable.
